Octowiz-v2 Onboarding

This guide takes you from a fresh machine to a working Octowiz flow using Podman + Zellij + OpenCode + the Octowiz CLI + Nuxt dashboard.

Goal: create a room, run a task pipeline, and inspect it from both CLI and web UI.

Table of Contents

  1. Install prerequisites
  2. Bootstrap the workspace
  3. Initialize and start Podman machine
  4. Configure model/runtime environment

  7. Daily workflow commands

System expectations

  • GitHub CLI auth is required for delivery/PR steps.
  • Podman must be running (sandbox provider defaults to Podman).
  • Zellij + OpenCode are used to launch collaborative coding sessions.
  • For agent/review/escalation paths, configure LiteLLM env vars or a local model command.

1Install prerequisites

If you already have these tools, skip to verification.

Install via Homebrew

brew install pnpm gh zellij podman
# OpenCode install method may vary by your setup; ensure `opencode` is on PATH.

Verify tool availability

pnpm --version
gh --version
zellij --version
podman --version
opencode --version

2Bootstrap the workspace

git clone https://github.com/raelli/octowiz-v2.git
cd octowiz-v2
pnpm install

Run baseline quality checks

pnpm lint
pnpm -r type-check
pnpm -r test

3Initialize and start Podman machine

podman machine list
podman machine init        # only needed once if no machine exists
podman machine start
podman info

On macOS, Podman needs this VM for containers. Keep it running while using Octowiz commands that start sandboxes.

4Configure model/runtime environment

Octowiz can run with LiteLLM-backed models (recommended path here) or a local model binary fallback.

Option A (LiteLLM path)

export LITELLM_BASE_URL="{{LITELLM_BASE_URL}}"
export LITELLM_API_KEY="{{LITELLM_API_KEY}}"

Option B (local model fallback)

export OCTOWIZ_MODEL_CMD="{{OCTOWIZ_MODEL_CMD}}"
Use Option A if you want router/gateway/aelli-backed flows. Use Option B only when running local model worker CLI.

GitHub auth check

gh auth status || gh auth login

5Run your first room and task

Create room + task

# create room
pnpm --filter @octowiz/cli octowiz create-room --name "Demo Room"

# capture returned room id, then create task
pnpm --filter @octowiz/cli octowiz create-task --room <ROOM_ID> --title "Implement onboarding slice"

Start room runtime (sandbox + zellij + opencode)

pnpm --filter @octowiz/cli octowiz start --room <ROOM_ID> --repo "$PWD"

Attach to the Zellij session

zellij attach octowiz-<ROOM_ID>

Run full orchestration pipeline

Before this step, ensure your local checkout is clean (git status --short) so branch/PR flow is predictable.

pnpm --filter @octowiz/cli octowiz run-task \
  --room <ROOM_ID> \
  --task <TASK_ID> \
  --agent impl-1 \
  --reviewer rev-1 \
  --repo "$PWD" \
  --branch feat/demo-room-task \
  --base main \
  --verdict approved

Inspect current room state

pnpm --filter @octowiz/cli octowiz status --room <ROOM_ID>

6Open the web dashboard

export NUXT_PUBLIC_ROOM_ID=<ROOM_ID>
export NUXT_LEDGER_DIR=".octowiz/ledger"
pnpm --filter @octowiz/web dev

Then open the local Nuxt URL and verify room status, participants, tasks, validations, reviews, and escalations.

7Daily workflow cheat sheet

# quick room startup (create + start)
pnpm --filter @octowiz/cli octowiz up --name "Daily Room" --repo "$PWD"

# assign task
pnpm --filter @octowiz/cli octowiz assign --room <ROOM_ID> --task <TASK_ID> --agent impl-1

# validate explicitly
pnpm --filter @octowiz/cli octowiz validate --room <ROOM_ID> --task <TASK_ID>

# review explicitly
pnpm --filter @octowiz/cli octowiz review --room <ROOM_ID> --task <TASK_ID> --reviewer rev-1 --verdict approved --repo "$PWD"

# escalate if needed
pnpm --filter @octowiz/cli octowiz escalate --room <ROOM_ID> --task <TASK_ID>

8Troubleshooting

Podman issues

podman machine list
podman machine start
podman info

Zellij session not found

zellij list-sessions -n -s
pnpm --filter @octowiz/cli octowiz start --room <ROOM_ID> --repo "$PWD"

OpenCode launch failures

opencode --version
which opencode

GitHub PR delivery failures

gh auth status
git remote -v

Model path failures

# LiteLLM path
env | grep LITELLM

# local model path
which octowiz-model
echo $OCTOWIZ_MODEL_CMD

If model env is missing, commands that need agent output/review/escalation will fail.

Optional smoke checks

# CLI smoke (requires OCTOWIZ_SMOKE + zellij binary)
OCTOWIZ_SMOKE=1 pnpm --filter @octowiz/cli test

# sandbox runtime smoke (requires OCTOWIZ_SANDBOX_SMOKE + running container runtime)
OCTOWIZ_SANDBOX_SMOKE=1 pnpm --filter @octowiz/sandbox-runtime test
You are ready when you can run create-room → start → run-task and see state in both CLI + dashboard.
